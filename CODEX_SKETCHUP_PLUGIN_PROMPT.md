# Codex Prompt for SketchUp Plugin Projects

Считай этот файл рабочей инструкцией для проекта. Перед любыми изменениями сначала прочитай его и следуй ему, если пользователь не сказал иначе.

## Главная цель

Мы делаем Ruby-плагины для SketchUp так, чтобы их было удобно быстро править и сразу тестировать.

## Как работать в этом проекте

1. Сначала изучи структуру текущего проекта.
2. Если для проекта ещё не настроен dev-loader, настрой его.
3. Не заставляй пользователя переустанавливать `.rbz` после каждой правки.
4. Любые обычные изменения должны тестироваться через локальные исходники проекта.
5. `.rbz` собирай только как дополнительный артефакт, а не как основной цикл разработки.

## Обязательный dev workflow

Нужен режим, при котором SketchUp грузит плагин прямо из рабочей папки проекта.

Цель:
- пользователь один раз подключает dev-loader
- после этого перезапускает SketchUp один раз
- дальше после правок достаточно нажать `Reload Plugin` в UI плагина или выполнить локальный reload-способ
- повторная установка через `Extension Manager` не нужна

Если dev-loader ещё не сделан, нужно:
- определить папку `SketchUp 20xx/SketchUp/Plugins`
- сделать loader `.rb` в папке Plugins, который грузит исходник из текущего проекта через `load`
- по возможности сохранить backup уже установленной версии
- добавить scripts для установки dev-loader и возврата к установленной версии

## Ожидаемое поведение от Codex

- Не предлагай каждый раз удалять старый плагин и ставить новый `.rbz`.
- Если можно просто поправить локальные исходники и reloadнуть, используй именно этот путь.
- После изменений кратко пиши, что именно проверить в SketchUp.
- Если для проверки нужен restart SketchUp, говори об этом явно.
- Если reload без рестарта достаточен, говори это явно.

## Что делать при изменениях логики

- Старайся не ломать существующий dev workflow.
- Если меняешь структуру загрузки файлов, проверь, что reload по-прежнему возможен.
- Если добавляешь новые Ruby-файлы, подключи их так, чтобы `load` мог подхватить обновления.

## Что должно быть в проекте

Желательно иметь:
- основной loader для SketchUp extension
- runtime-файлы плагина
- `build` script для `.rbz`
- `install_dev_loader` script
- `restore_installed_plugin` script
- если есть UI, кнопку `Reload Plugin` внутри окна

## Формат ответа после правок

После работы дай короткий practical summary:
- какие файлы изменены
- нужен ли restart SketchUp
- если есть новый `.rbz`, где он лежит

## Если проект новый

Если папка почти пустая, можно создавать структуру с нуля, но всё равно сразу закладывай удобный dev workflow, а не только упаковку в `.rbz`.

## Шаблон команды для пользователя

Если пользователь говорит:

`прочитай CODEX_SKETCHUP_PLUGIN_PROMPT.md и работай по нему`

то сначала прочитай этот файл, потом продолжай работу уже с учётом этих правил.
